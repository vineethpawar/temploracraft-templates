# Contributing a template

Everything in this repo is community-authored. Four paths to submit — pick the one that matches how comfortable you are with GitHub.

## Level 1 — one-click Publish (recommended)

1. Open [app.temploracraft.com/templates](https://app.temploracraft.com/templates).
2. Click **Contribute a template**.
3. Design your template in the canvas author tool. Drag text, images, dividers, section containers. Drop **placeholder** nodes (`{{firstName}}`, `{{experience.title}}`, …) wherever you want data to fill in.
4. Click **Publish**. You'll be asked to sign in with GitHub — this gives the app a scoped `public_repo` OAuth token so it can open the PR on your behalf.
5. Fill in slug + display name + one-line description. Click **Create pull request**.
6. Behind the scenes the app forks this repo into `<your-username>/temploracraft-templates`, creates a branch, commits `templates/resume/<slug>/template.json` + `README.md` + `preview.png`, and opens a PR.
7. CI runs. Green build → we merge → your template appears on the main site on next redeploy (usually within a day).

## Level 2 — "Open in GitHub" (no OAuth)

If you'd rather not connect GitHub to the app:

1. Design your template in the canvas author tool.
2. Click **Publish → Open in GitHub**.
3. GitHub opens with the `template.json` file pre-populated in the file editor. Click **Propose new file** — GitHub auto-forks and creates the PR.
4. Download the `preview.png` from the modal and drag it into the same PR as a separate commit.

## Level 3 — download the bundle

For the fully-offline path:

1. Design your template.
2. Click **Publish → Download bundle**. You get a ZIP with `template.json`, `README.md`, `preview.png`, and `HOW-TO-SUBMIT.md`.
3. Follow the four-step visual guide inside the ZIP: fork this repo on github.com, drop the files under `templates/resume/<slug>/`, open a PR.

## Level 4 — Claude Code users

If you use [Claude Code](https://claude.com/product/claude-code) locally, the Publish modal offers a copy-to-clipboard prompt like:

> `Please open a PR against github.com/vineethpawar/temploracraft-templates. Add the three attached files at templates/resume/<slug>/. PR title: 'Add <Name> template'. PR body: <auto-generated summary>.`

Paste that into your Claude Code terminal with the ZIP files attached. Claude uses your local git config to open the PR from your GitHub identity. No temploracraft-side infrastructure involved.

## What CI checks

Every PR to `main` runs a validation job:

- ✅ `template.json` parses against the [Zod schema](packages/schema/src/zod.ts).
- ✅ Every `bind` on a `placeholder-text` node references a real field on the current [`ResumeData` schema](https://github.com/vineethpawar/temploracraft/blob/main/apps/web/src/domains/resume/schema.ts).
- ✅ `preview.png` is present, under 200 KB, matches an 816×1056 aspect (± 2px).
- ✅ `README.md` present with the frontmatter (name, author, license: MIT).
- ✅ No external URLs in image nodes (base64 data URIs only, or bundled assets referenced by relative path).

Failing checks post a review comment describing what to fix.

## Style guide

- **Slug**: kebab-case, letters + digits + hyphens only. Examples: `modern-serif`, `minimal-two-column`.
- **Preview**: your `preview.png` should show the template filled with realistic sample data — recruiters browsing the gallery are the audience.
- **Placeholders**: prefer standard field names. If you invent a new field, open a discussion first — the main app has to know about it.
- **License**: MIT for every submission. If you'd prefer another license, open an issue and let's talk.

## Attribution

Every template's `template.json` has an `author` block:

```json
"author": {
  "name": "Your Name",
  "url": "https://your-site.com",
  "githubUsername": "your-github"
}
```

That's what shows up in the gallery card and on the `/templates/<slug>` detail page. Your name, forever, next to your work.
